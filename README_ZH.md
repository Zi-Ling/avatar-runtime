# avatar-runtime

一个用于 **执行 LLM 生成结构化计划** 的  
**最小执行运行时（execution runtime）**。

本项目专注于 **确定性执行**、**快速失败（fail-fast）校验** 以及  
**基于策略的动作约束** ——  
**不**做规划、不做记忆、不做智能。

当一个执行计划存在信息缺失、不安全或语义不明确时，  
该运行时 **不猜测、不自动修复、也不悄悄重试**。

---

## 为什么要做这个项目

在真实的 Agent 系统中，大多数失败并不是模型能力不足导致的。

问题往往出现在 **执行层（execution layer）**：

- 参数缺失或结构错误
- 执行了不安全或越权的动作
- 隐式重试掩盖了真实错误
- 执行过程不可回放、不可审计

**avatar-runtime** 将这些问题单独拆出来，  
通过 **严格的执行语义**，在“计划”与“真实世界”之间建立一道明确的边界。

---

## 这个项目是什么

- 一个确定性的执行运行时
- 一个对结构化计划进行 fail-fast 校验的执行器
- 一个用于阻断危险操作的策略层
- 一个可追踪、可回放的执行循环

## 这个项目不是什么

- 不是 Agent 框架
- 不是 Planner 或任务拆解系统
- 不是记忆或上下文系统
- 不是自动化工具集
- 不是 LLM 封装器

本项目 **刻意回避“智能”**，只关注一件事：  
**执行是否正确、是否安全、是否可验证**。

该运行时 **用于作为系统中的执行层被嵌入使用**，  
而 **不是** 面向终端用户直接使用的工具。

---

## 它在系统中的位置

本仓库刻意只展示一个更大 Agent 系统中的  
**单一执行切片**。

该执行运行时通常位于系统中  
**规划之后**、**真实副作用之前**。

在完整系统中，该执行层通常会接收：

- 由 LLM 或 Planner 生成的结构化执行计划
- 从上下文中解析出的明确参数
- 由宿主系统定义的严格执行策略

本演示刻意省略这些上游组件，  
以保证执行行为 **可观察、可测试、且具备确定性**。

---

## 快速开始

克隆仓库并运行演示：

    python run_demo.py

演示包含三个执行场景：

1. **成功执行**  
   一个简单的文件操作流程，完整执行并产生结果。

2. **参数校验失败**  
   当执行步骤缺少必需参数时，在真正执行前立即失败。

3. **策略阻断**  
   尝试执行危险操作时，被执行策略明确拒绝。

每一次运行都会生成 **可回放的执行轨迹（trace artifacts）**，位于：

    ./workspace/artifacts/

---

## 演示行为说明

- 清晰展示每一步的执行顺序
- 在产生副作用前完成参数与结构校验
- 明确说明策略拒绝的原因
- 为每一次执行生成确定性的 trace 文件

示例文件：

    trace_success.json
    trace_missing_param.json
    trace_policy_block.json

这些 trace 并非普通日志，  
而是用于 **检查、回放和分析** 的执行证据。

---

## 设计原则

- **快速失败**：输入不合法时立即终止
- **拒绝隐式修复**：不猜、不补、不兜底
- **执行前策略校验**：危险动作永远不会被执行
- **确定性轨迹**：每次执行都可审计、可复现

## 项目结构

    runtime/        执行引擎、校验、策略、追踪
    skills/         最小内置技能（仅文件相关）
    examples/       演示计划与运行入口
    run_demo.py     顶层演示入口
    workspace/      沙箱执行目录（不纳入版本控制）

项目结构刻意保持克制与显式。
